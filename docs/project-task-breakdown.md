# Project Task Breakdown - Capstone Management System

## Project Overview
**System Name:** Project Workspace - BukSU Capstone Management System
**Target Users:** 37+ respondents (BSIT/BSEMC 3rd & 4th Year Students)
**Technology Stack:** MERN (MongoDB, Express.js, React with Vite, Node.js)
**Evaluation Framework:** ISO 25010 Quality Model
**Estimated Timeline:** 5 Phases (18-24 weeks)

### 2.2 Database Schema Design
**Objective:** Design MongoDB database structure

#### Task 2.2.1: Create Entity-Relationship Diagram (ERD)
- **Description:** Design database entities and relationships
- **Deliverables:**
  - ERD diagram (using draw.io or Lucidchart)
  - Entity descriptions
  - Relationship cardinality documentation
- **Collections:**
  - `users` - User accounts
  - `projects` - Capstone projects
  - `documents` - Document metadata
  - `submissions` - Version-controlled submissions
  - `comments` - Feedback and comments
  - `notifications` - System notifications
  - `audit_logs` - Activity logs
- **Time Estimate:** 2 days
- **Acceptance Criteria:**
  - All entities identified
  - Relationships properly defined
  - Normalization validated

#### Task 2.2.2: Define Mongoose Schemas
- **Description:** Create Mongoose schema definitions for all collections
- **Deliverables:**
  - Schema files for each model (`server/models/`)
  - Field validation rules
  - Index strategy
- **Schema Details:**
  - **User Schema:**
    - `firstName`, `lastName`, `email`, `password` (hashed)
    - `role` (enum: student, adviser, panel, coordinator)
    - `studentId`, `department`, `yearLevel`
    - `isVerified`, `createdAt`, `updatedAt`
  - **Project Schema:**
    - `title`, `description`, `status` (enum: draft, submitted, under_review, approved, rejected)
    - `students[]` (array of user IDs)
    - `adviser` (user ID)
    - `panelMembers[]` (array of user IDs)
    - `submittedAt`, `approvedAt`
  - **Document Schema:**
    - `projectId`, `fileName`, `fileType`, `fileSize`
    - `googleDriveFileId`, `googleDriveUrl`
    - `version`, `uploadedBy`, `uploadedAt`
  - **Submission Schema:**
    - `projectId`, `documentId`, `submittedBy`
    - `version`, `status`, `plagiarismReportUrl` (placeholder)
    - `comments`, `submittedAt`
- **Time Estimate:** 3 days
- **Acceptance Criteria:**
  - All schemas implemented
  - Validation rules applied
  - Indexes defined for query optimization

#### Task 2.2.3: Define Database Indexes
- **Description:** Identify frequently queried fields and create indexes
- **Deliverables:**
  - Index strategy document
  - Performance benchmarking plan
- **Indexes:**
  - `users`: `email` (unique), `studentId` (unique)
  - `projects`: `title`, `status`, `adviser`, `students`
  - `documents`: `projectId`, `googleDriveFileId`
  - `submissions`: `projectId`, `status`, `submittedAt`
- **Time Estimate:** 1 day
- **Acceptance Criteria:**
  - Indexes created for all critical queries
  - Query performance tested
  - No over-indexing
